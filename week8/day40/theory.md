# День 40 — Практика модели объектов

## 1. Объект и ссылка

Переменная в Python хранит **ссылку**, а не значение.

```python
a = [1, 2]
b = a
```

Создаётся один список. Две переменные указывают на один объект.

Изменение по месту (`append`, `extend`, `sort`) меняет сам объект.

## 2. Изменение vs переприсваивание

```python
a[0].append(99)   # изменение объекта

a[1] = ["y"]     # переприсваивание ссылки
```

- Изменение влияет на все ссылки.
- Переприсваивание влияет только на текущий контейнер.

## 3. Shallow copy

```python
b = a[:]
b = copy.copy(a)
```

Создаётся новый внешний контейнер.
Внутренние объекты остаются теми же.

Следствие: изменение вложенного объекта отражается в обоих списках.

## 4. Deep copy

```python
c = copy.deepcopy(a)
```

Создаются новые контейнеры на всех уровнях.
Вложенные объекты больше не разделяются.

## 5. `is` vs `==`

- `is` — проверка идентичности (один ли объект)
- `==` — проверка равенства значений

Вложенные структуры могут быть:

- разными объектами, но равными (`== True`, `is False`)
- одним объектом (`== True`, `is True`)

## 6. Типовые ошибки, выявленные в практике

1. Игнорирование мутации объекта (забыть, что `append` меняет список).
2. Путаница между строками и вложенными списками при анализе структуры.
3. Неправильное предположение о типе ошибки (`IndexError`, а не `ReferenceError`).
4. Ожидание, что shallow copy «копирует всё».

## 7. Ментальная модель

- Контейнер хранит ссылки.
- Shallow copy копирует контейнер, но не вложенные объекты.
- Deep copy копирует всю структуру.
- Мутация и переприсваивание — разные операции.

Если это различие автоматически читается в коде — модель усвоена.
