# День 36 — Всё объект. Идентичность и равенство

## 1. Всё — объект

В Python любое значение является объектом:

- числа (`int`, `float`)
- строки (`str`)
- списки (`list`)
- функции
- классы

У каждого объекта есть:

- тип
- идентичность
- значение

Переменная хранит не «значение», а ссылку на объект.

```python
x = 10
y = x
```

Теперь `x` и `y` указывают на один и тот же объект.

---

## 2. Идентичность (`is`) и равенство (`==`)

`==` сравнивает значения.

```python
[1, 2] == [1, 2]  # True
```

`is` сравнивает идентичность — это один и тот же объект или нет.

```python
a = [1, 2]
b = [1, 2]

a is b  # False
```

Это разные объекты с одинаковым содержимым.

---

## 3. Функция `id()`

`id(obj)` возвращает идентификатор объекта (в CPython — адрес в памяти).

```python
a = 5
b = a

id(a) == id(b)  # True
```

---

## 4. Singleton-объекты

Некоторые объекты существуют в единственном экземпляре:

- `None`
- `True`
- `False`

Для них корректно использовать `is`.

```python
x = None
x is None  # True
```

---

## 5. Почему нельзя использовать `is` для чисел

В CPython маленькие числа (обычно от −5 до 256) кэшируются.

```python
a = 256
b = 256
a is b  # может быть True

c = 1000
d = 1000
c is d  # может быть False
```

Это деталь реализации, а не гарантия языка.

Правило:

- для значений использовать `==`
- `is` — только для `None` и singleton-объектов

---

## 6. `== True` и `is True`

`x == True` — проверка равенства значению.

`x is True` — проверка, что объект — именно singleton `True`.

Идиоматично писать просто:

```python
if x:
    ...
```

Это проверка truthy-значения без лишнего сравнения.
