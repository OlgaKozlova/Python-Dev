```python
a = [1, 2]
b = a
a = a + [3]
print(a)
print(b)
```

## Задача 2

Что выведет код:


Объяснить через механизм `__add__` / `__iadd__`.

## Задача 3

Что выведет:

Почему так происходит?

## Задача 5

Что произойдёт:

```python
def f(x):
    x += [1]


a = [0]
f(a)
print(a)
```

И почему?

---

# Решения

## Решение 1

`a + [3]` создаёт новый объект списка.

`a` перепривязывается к новому объекту.

`b` остаётся ссылкой на старый список.

Вывод:

```
[1, 2, 3]
[1, 2]
```

---

## Решение 2

`+=` для списка вызывает `__iadd__`.

Список мутируется in-place.

Обе переменные указывают на один объект.

Вывод:

```
[1, 2, 3]
[1, 2, 3]
```

---

## Решение 3

`[[0]] * 3` копирует ссылки на один вложенный список.

Мутация одного изменяет все.

Вывод:

```
[[0, 1], [0, 1], [0, 1]]
```

---

## Решение 4

List comprehension создаёт три независимых вложенных списка.

Изменяется только первый.

```
[[0, 1], [0], [0]]
```

---

## Решение 5

`+=` вызывает `__iadd__` у списка.

Происходит мутация переданного объекта.

```
[0, 1]
```
