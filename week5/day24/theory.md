# День 24 — Как Python ищет модули (sys.path)

## Ключевая идея

Python **не знает «корень проекта»**. Он ищет модули, проходя по списку путей `sys.path` сверху вниз и берёт **первое совпадение**.

---

## sys.path — что это

`sys.path` — список директорий, где Python ищет модули.

Типичный состав:

1. **Путь запуска** (самый важный)
2. Стандартная библиотека
3. `site-packages`

```python
import sys
print(sys.path)
```

---

## Главное различие запусков

### 1. `python file.py`

- Python запускает **файл по пути**
- `sys.path[0]` = папка, где лежит файл
- папки выше **не видны**

### 2. `python -m pkg.mod`

- Python запускает **модуль по имени**
- `sys.path[0]` = текущая рабочая директория
- пакетная модель импорта **включена**

Ключевая формула:

> запуск по пути → видна папка файла
> запуск по имени → видна текущая папка

---

## Почему это важно

- один и тот же файл может работать или падать в зависимости от способа запуска
- импорты ломаются не из-за кода, а из-за контекста

---

## **init**.py

- начиная с Python 3.3 папка **может быть пакетом без `__init__.py`** (namespace package)
- это работает при запуске через `-m`
- в реальных проектах `__init__.py` кладут **явно**, для предсказуемости

---

## Что НЕ делать

- не использовать `PYTHONPATH` для архитектуры
- не писать `sys.path.append()` в коде
- не запускать вложенные файлы напрямую
