# День 24 — Практика

## Цель

Руками увидеть, как **способ запуска меняет контекст импорта**.

---

## Структура

Создай папку `day24`:

```
day24/
├── a.py
└── sub/
    └── b.py
```

---

## Код

### a.py

```python
print("a imported")
```

### sub/b.py

```python
import sys

print("sys.path:")
for p in sys.path:
    print(p)

import a
```

---

## Запуск 1 (ожидается ошибка)

Из папки `day24`:

```bash
python sub/b.py
```

Что должно произойти:

- `ModuleNotFoundError: No module named 'a'`

Почему:

- `sys.path[0]` указывает на `sub/`
- `a.py` лежит уровнем выше и **невидим**

---

## Запуск 2 (работает)

Из папки `day24`:

```bash
python -m sub.b
```

Что должно произойти:

- вывод `sys.path`
- сообщение `a imported`

Почему:

- `sys.path[0]` = текущая директория (`day24`)
- `a.py` найден корректно
