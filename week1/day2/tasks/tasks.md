# День 2 — Практика: задачи на объекты, ссылки и namespace

Ниже — **4–5 задач-ловушек** на материал Дня 2.
Цель — проверить **понимание модели Python**, а не синтаксис.

❗ Решай **мысленно или в интерпретаторе**, обязательно объясняя *почему* получился именно такой результат.

## Задача 1 — Перепривязка vs мутация

```python
def f(x):
    x = x + [4]

lst = [1, 2, 3]
f(lst)
print(lst)
```

**Вопросы:**

1. Что выведется?
2. Был ли создан новый объект?
3. Менялся ли исходный список?

---

## Задача 2 — `+=` как ловушка

```python
def f(x):
    x += [4]

lst = [1, 2, 3]
f(lst)
print(lst)
```

**Вопросы:**

1. Чем эта задача отличается от предыдущей?
2. Почему результат другой?
3. Какой метод объекта был вызван?

---

## Задача 3 — Срез и идентичность

```python
a = [1, 2, 3]
b = a

b[:] = [9, 9]

print(a)
print(a is b)
```

**Вопросы:**

1. Почему изменился `a`?
2. Изменился ли объект?
3. Изменилась ли ссылка?

---

## Задача 4 — Namespace функции и модуля

```python
x = 10

def f():
    x = 20

f()
print(x)
```

**Вопросы:**

1. Какой `x` был изменён?
2. Почему глобальный `x` остался прежним?
3. В каком namespace живёт каждый `x`?

---

## Задача 5 — Поиск имён (LEGB)

```python
x = 1

def outer():
    x = 2
    def inner():
        print(x)
    inner()

outer()
```

**Вопросы:**

1. Что выведется?
2. В каком namespace найдено имя `x`?
3. Почему не используется глобальный `x`?

---

## Как считать задачи решёнными

Задача считается **усвоенной**, если ты можешь:

* назвать результат
* объяснить его **через объекты и ссылки**
* указать, **менялся объект или имя**
* (для namespace) назвать область видимости

Если хотя бы в одной задаче ответ был «интуитивный, но неясный» — стоит вернуться к теории.
