## День 1 — основы (кратко, структурировано, без лишнего)

### 1. Что такое Python

**Python** — это:
- интерпретируемый язык общего назначения
- ориентирован на читаемость и простоту
- выполняется построчно, без предварительной компиляции в бинарник

**Ключевая идея Лутца:**  
Python — не язык «для новичков», а язык для быстрой и надёжной разработки.

---

### 2. Интерпретатор

Интерпретатор Python:
- читает исходный код `.py`
- преобразует его во внутренний байткод
- немедленно выполняет

Важно:
- нет шага компиляции как в C/C++  
- ошибки обнаруживаются во время выполнения

---

### 3. Выполнение `.py` файлов

`.py` — обычный текстовый файл.

Запуск:
```bash
python file.py
```

Происходит:
- файл читается сверху вниз
- каждая строка выполняется по порядку
- `def` и `class` не выполняются, а регистрируются

Следствие: порядок кода критичен.

---

### 4. REPL (интерактивный режим)

**REPL = Read → Eval → Print → Loop**

Запуск:
```bash
python
```

Особенности:
- каждая строка выполняется сразу
- результат выражения выводится автоматически
- удобен для экспериментов и проверки идей

По Лутцу:  
REPL — инструмент мышления, а не просто консоль.

---

### 5. `print` и простые выражения

Примеры:
```python
print("Hello")
print(2 + 3)
```

Простые выражения:
```python
2 + 3
"a" * 3
10 > 5
```

- в REPL результат выражения показывается автоматически
- в `.py` файле — только при явном вызове `print`

---

### 6. `def` и `class`: описание, а не выполнение

#### `def` — что происходит при чтении файла

```python
def f():
    print("hi")
```

При выполнении файла:

- тело функции **не выполняется**
- Python:
  - создаёт объект функции
  - связывает имя `f` с этим объектом

Фактически:
```python
f = <function object>
```

Выполнение происходит только при вызове:
```python
f()
```

---

#### `class` — определение класса

```python
class A:
    x = 10
    print("inside class")
```

При выполнении файла:

- тело класса выполняется **один раз**
- это нужно для:
  - создания атрибутов
  - формирования объекта класса
- методы **не выполняются**
- `print` внутри класса сработает при определении класса

---

#### Сравнение конструкций

| Конструкция | Поведение при чтении файла |
|------------|----------------------------|
| `print()`  | выполняется сразу          |
| `x = 2+3`  | вычисляется сразу          |
| `def f()`  | создаёт функцию            |
| `class A`  | создаёт класс              |

`def` и `class` не исполняют логику программы —  
они описывают объекты, которые будут использоваться позже.

---

#### Почему это критично

```python
f()
def f():
    pass
```

Ошибка:
- имя `f` ещё не существует

Правильно:
```python
def f():
    pass

f()
```

---

### Итог дня (по Лутцу)

Ты должен понимать:

- Python исполняет, а не компилирует
- код читается строго сверху вниз
- `def` и `class` создают объекты, а не выполняют логику
- REPL и `.py` — два режима одной системы
- выражение ≠ вывод (кроме REPL)

Это фундамент. Без него дальше книга не складывается.
