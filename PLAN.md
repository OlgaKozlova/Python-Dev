# План перехода из фронтенда в фулстек  
**Python + Django | обучение без курсов | один сквозной проект**

---

## Цель
Перейти от узкопрофильного фронтенд-разработчика к **фулстеку (frontend + backend)**  
через **Python и Django**, с упором на практику и реальный проект.

---

## Входные условия
- Опытный фронтенд-разработчик  
- Время: **~1 час в день, 5–6 дней в неделю**  
- Формат: обучение + один сквозной проект  
- Курсы не используются  

---

## Формат работы
- Один проект на всё обучение  
- Каждая тема сразу применяется в коде  
- Теория — только под текущую задачу  
- Фокус на архитектуре и мышлении, а не на «зазубривании»  

---

## Сквозной проект
**Backend для пользовательского сервиса**

Базовые сущности:
- User  
- Profile  
- Item (пост / объект / запись)  
- Comment (опционально)

Проект универсальный: может быть адаптирован под форум, сервис, админку, клуб, API для SPA.

---

## Учебный план по неделям

---

### Неделя 1 — Python как рабочий инструмент
**Фокус:** уверенно писать прикладной код.

Темы:
- функции и области видимости  
- структуры данных (dict / list / set) в прикладном контексте  
- dataclass  
- typing (минимум, но корректно)  
- исключения  
- работа с датами  
- env-переменные  

**Результат:**  
уверенное владение Python для серверных задач.

---

### Неделя 2 — Веб-основа и старт Django
**Фокус:** убрать ощущение «магии».

Темы:
- HTTP, методы, статусы  
- REST — базовые принципы  
- структура Django-проекта  
- apps  
- urls  
- views  
- разбор ключевых настроек в `settings.py`  

**Результат:**  
Django-приложение с работающим JSON-эндпоинтом.

---

### Неделя 3 — Модели и база данных
**Фокус:** данные как основа продукта.

Темы:
- ORM как абстракция  
- модели Django  
- связи (OneToOne, ForeignKey, ManyToMany)  
- миграции  
- базовое понимание оптимизации запросов  

**Результат:**  
реальная схема данных, не «учебная».

---

### Неделя 4 — API (Django REST Framework)
**Фокус:** ценность для фронтенда.

Темы:
- Django REST Framework  
- serializers  
- viewsets  
- permissions  
- обработка ошибок  
- пагинация и фильтрация  

**Результат:**  
полноценный CRUD API для SPA.

---

### Неделя 5 — Авторизация и безопасность
**Фокус:** реальные пользовательские сценарии.

Темы:
- пользователи  
- регистрация и логин  
- JWT или session-based auth  
- защита эндпоинтов  
- CORS  

**Результат:**  
безопасный backend, готовый к использованию.

---

### Неделя 6 — Архитектура и качество кода
**Фокус:** код, который не стыдно показывать.

Темы:
- структура проекта  
- разделение ответственности  
- сервисный слой  
- базовые тесты  
- логирование  

**Результат:**  
поддерживаемый и понятный backend.

---

### Неделя 7–8 (опционально)
**Фокус:** приближение к продакшену.

Темы:
- деплой  
- окружения  
- интеграция с фронтом  
- рефакторинг  

**Результат:**  
осознание полного жизненного цикла backend-приложения.
