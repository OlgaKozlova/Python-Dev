# День 26 — Исключения в Python

## Что такое исключение

Исключение — это объект, сигнализирующий о нештатной ситуации во время выполнения программы. В Python ошибки **не возвращаются значениями**, а **выбрасываются** и поднимаются по стеку вызовов.

Если исключение не перехвачено, интерпретатор:

- печатает traceback
- завершает выполнение программы

## try / except — базовая форма

```python
try:
    risky_operation()
except ValueError:
    handle_error()
```

Смысл конструкции:

- `try` — код, где _может_ произойти ошибка
- `except` — реакция **на конкретный тип** ошибки

Важно: `except` выполняется **только если** в `try` произошло исключение указанного типа.

## Исключения — не if

`try / except` **не предназначен** для управления логикой.

Плохо:

```python
try:
    result = data[key]
except KeyError:
    result = None
```

Лучше:

```python
result = data.get(key)
```

Используй исключения только тогда, когда:

- ошибка действительно исключительная
- альтернативной проверки нет или она дороже/хуже

## Ограничение try-блока

Правило:

> В `try` должен быть только код, который реально может упасть

Плохо:

```python
try:
    x = int(value)
    y = x + 10
    print(y)
except ValueError:
    ...
```

Хорошо:

```python
try:
    x = int(value)
except ValueError:
    ...

y = x + 10
print(y)
```

## Как поднимается исключение

- Исключение возникает в текущей функции
- Если не перехвачено — поднимается выше
- Так до точки входа (`__main__`)

Это объясняет, почему ошибки могут «вылетать» не там, где написан `try`.

## Итог

- Исключения — часть модели исполнения Python
- `try / except` — механизм обработки сбоев
- Ловить нужно **конкретные** типы ошибок
- Исключения не заменяют условия
