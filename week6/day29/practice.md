## Задача 1

Есть функция:

```python
def load_number(path: str) -> int:
    with open(path) as f:
        return int(f.read())
```

**Вопрос:**

- Где здесь _нельзя_ ловить исключения?
- Где их _нужно_ ловить?

Опиши словами, без кода.

## Задача 2

Посмотри на код:

```python
def process(values):
    result = []
    for v in values:
        try:
            result.append(int(v))
        except Exception:
            pass
    return result
```

**Задание:**

1. Объясни, какие ошибки здесь замаскированы
2. Почему это плохой код
3. Какой уровень ответственности нарушен

---

## Задача 3

Есть уровни:

- `parse()` — утилита
- `handle_request()` — бизнес-логика
- `main()` — точка входа

**Задание:**
Распредели, на каком уровне:

- не ловят исключения
- оборачивают
- ловят и обрабатывают

---

## Решения (после твоего ответа)

### Задача 1

- `load_number` — низкий уровень, не ловит
- ловить нужно в точке входа, где можно решить: сообщение, retry, fallback

### Задача 2

- маскируются `ValueError`, `TypeError`, `OSError`
- теряется информация о данных и причине ошибки
- нарушен принцип уровней ответственности

### Задача 3

- `parse()` — не ловит
- `handle_request()` — может оборачивать
- `main()` — ловит и принимает решение
