# День 11 — if / elif / else, логические операторы, truthy / falsy

## Теория

### 1. `if / elif / else`

`if` в Python — это оператор управления потоком, а не выражение. Он **не возвращает значение**.

```python
if x > 0:
    y = 1
elif x == 0:
    y = 0
else:
    y = -1
```

- Проверки идут сверху вниз.
- Выполняется ровно одна ветка.
- `elif` — часть одной конструкции, а не вложенный `if`.

Для выражений используется тернарный оператор:

```python
y = 1 if x > 0 else -1
```

---

### 2. Truthy / falsy

В `if` объекты **неявно приводятся** к логическому значению.

Falsy значения:

- `False`
- `None`
- `0`, `0.0`
- `""`
- `[]`, `{}`, `set()`

Всё остальное — truthy.

Важно:

```python
x is None      # правильно
x == None      # неправильно
```

---

### 3. `and`, `or`, `not`

`and` и `or` возвращают **один из операндов**, а не `bool`.

```python
"" or "default"     # "default"
0 and 10               # 0
10 and 20              # 20
```

Правила:

- `and` → первый falsy или последний
- `or` → первый truthy или последний

`not` всегда возвращает `bool`.

---

### 4. Короткое замыкание

Правая часть выражения может **не выполняться**, если результат уже известен.

```python
0 and (1 / 0)      # деление не выполнится
"ok" or (1 / 0)   # деление не выполнится
```

Используется для:

- безопасных проверок (`None`)
- контроля побочных эффектов
- уменьшения количества `if`
