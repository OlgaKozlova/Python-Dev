# День 31 — Классы: class, **init**, self

## Класс как объект

В Python `class` — это объект, создаваемый во время выполнения. Он является экземпляром `type` и служит фабрикой для создания других объектов — экземпляров класса.

```python
class User:
    pass
```

```python
u = User()
```

`User` — объект, `u` — экземпляр.

---

## **init**

`__init__` — инициализатор, а не конструктор. Объект уже создан к моменту его вызова.

```python
class User:
    def __init__(self, name):
        self.name = name
```

При вызове:

```python
u = User("Olga")
```

Фактически выполняется:

```python
User.__init__(u, "Olga")
```

---

## self

`self` — ссылка на конкретный экземпляр.

Важно:

- `self` — обычный параметр функции
- он **явно объявлен в сигнатуре метода**
- Python подставляет его автоматически при вызове через экземпляр

Эквивалентные вызовы:

```python
u.greet()
User.greet(u)
```

---

## Атрибуты экземпляра

Атрибут экземпляра создаётся при присваивании:

```python
self.name = name
```

Фактически:

```python
u.__dict__ == {"name": "Olga"}
```

Если забыть `self`, переменная станет локальной и исчезнет после `__init__`.

---

## Ключевая мысль

- Класс — объект
- Метод — функция + привязка к объекту
- `self` — явная часть модели, а не скрытая магия
