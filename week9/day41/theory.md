# День 41 — Функции как объекты

## 1. Функция — объект первого класса

В Python функция является полноценным объектом. Это означает, что она:

- имеет тип (`function`)
- может быть присвоена переменной
- может передаваться как аргумент
- может возвращаться из другой функции
- может храниться в структурах данных

Пример:

```python
def greet(name):
    return f"Hello, {name}"

x = greet
print(x("Anna"))
```

`x` и `greet` ссылаются на один и тот же объект функции.

---

## 2. Объект функции vs вызов функции

Важно различать:

```python
greet      # объект функции
greet()    # результат вызова
```

Передача функции:

```python
def run(func):
    return func()

run(greet)      # передаём функцию
run(greet())    # передаём результат вызова
```

Во втором случае будет ошибка логики, так как передаётся не функция, а строка.

---

## 3. Хранение функций

Функции можно хранить в списках и словарях.

```python
def add(x, y): return x + y
def mul(x, y): return x * y

operations = [add, mul]

for op in operations:
    print(op(2, 3))
```

Это основа для:

- таблиц диспетчеризации
- стратегий
- плагинных архитектур

Пример словаря:

```python
def dispatch(op):
    def add(a, b): return a + b
    def mul(a, b): return a * b

    ops = {
        "add": add,
        "mul": mul,
    }

    try:
        return ops[op]
    except KeyError:
        raise ValueError("unknown op")
```

---

## 4. Порядок выполнения внутри функции

Тело функции выполняется последовательно во время вызова.

1. Создаются вложенные функции
2. Создаются структуры данных
3. Выполняется `return`

Если попытаться использовать имя до его связывания, будет `NameError`.

В Python нет hoisting (в отличие от JavaScript).

---

## 5. Связь с областью видимости

Область видимости определяется лексически (в момент определения функции),
но разрешение имён происходит во время вызова.

Python запоминает, **где искать имя**, но не «замораживает» его значение.

Это важно для понимания замыканий и late binding (следующие темы).
